# AI-Powered Intelligent Job Application Assistant

A full-stack MVP that helps users discover, evaluate, and prepare high-quality job applications with AI support while keeping human approval in the loop.

## Tech Stack

- Frontend: React + Vite
- Backend: FastAPI + SQLAlchemy
- AI Layer: OpenAI API (with safe fallback behavior)
- Database: SQLite by default (switchable to PostgreSQL via `DATABASE_URL`)

## Implemented Modules

1. Secure Authentication (JWT) (`/api/auth/register`, `/api/auth/login`, `/api/auth/me`)
2. Compliance Consent + Event Logging (`/api/compliance/consent`, `/api/compliance/events`)
3. Job Input + Parsing (`/api/jobs/parse`)
	- Supports manual text and uploaded TXT/PDF parsing (`/api/jobs/upload`)
4. Resume Upload Parsing (`/api/resume/upload`)
5. Job Matching Engine with weighted scoring (`/api/match`)
6. Resume Tailoring Engine (`/api/resume/tailor`)
7. Application Content Generator (`/api/content/generate`)
8. Decision Engine (`/api/decision`)
9. Assisted Application helper with platform-specific autofill payloads and checkpoint confirmation (`/api/autofill/prepare`, `/api/autofill/confirm`)
10. Application Tracker Dashboard (`/api/tracker`, `/api/analytics`)

## Compliance and Guardrails

- No auto-login or one-click auto-submission is implemented.
- User review and explicit confirmation checkpoints are required before final external application actions.
- One-click generation is available only for preparation: analysis, tailoring, content, and autofill payload generation.
- A final human review gate must be completed before marking a job ready for manual submit.
- Resume tailoring prompt enforces truthfulness and no fabricated experience.
- JWT-based auth is used for session security.
- Raw credential storage is prohibited; passwords are stored as salted hashes.

## Run Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
uvicorn app.main:app --reload --port 8000
```

## Run Frontend

```bash
cd frontend
npm install
npm run dev
```

## Browser Extension Scaffold (Manual Submit Safe)

A lightweight extension scaffold is included in `browser-extension/` for browser-assisted fill on LinkedIn, Indeed, and Naukri.

### Features

- Reads payload generated by `POST /api/autofill/prepare`
- Helps with field-by-field paste using the currently focused input field
- Blocks submit actions until explicit manual submit confirmation is checked
- Never automates login or final submission

### Load Extension (Chrome/Edge)

1. Open extension management page (`chrome://extensions` or `edge://extensions`)
2. Enable **Developer mode**
3. Click **Load unpacked** and select `browser-extension/`

### Usage

1. Generate autofill payload in app UI (or API).
2. Open target application page (LinkedIn/Indeed/Naukri).
3. Click extension icon and paste payload JSON into popup.
4. Click **Load Payload Into Page**.
5. In page helper panel, first try **Auto Fill Common Fields** (platform selector templates).
6. For remaining values, click target field then choose **Fill Next Field**.
7. Before submit, check manual confirmation in panel.
8. Complete final submission yourself.

You can tune selector matching from the extension popup using **Apply Selector Overrides** JSON.
